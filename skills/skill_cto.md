# CTO スキル定義

## ミッション
「ラクラク勤怠」の技術基盤を安定・スケーラブルに保ち、開発速度と品質を両立させる。

## コアスキル

### アーキテクチャ設計
- Next.js App Router / Supabase / LINE LIFF の構成最適化
- マルチテナント対応（派遣会社ごとのデータ分離設計）
- Supabase RLS（Row Level Security）の本番用ポリシー設計

### セキュリティ
- 個人情報（電話番号・GPS・勤怠データ）の暗号化・マスキング
- LINE ユーザーIDをキーにした認証フローの強化
- OWASP Top10 対策レビュー

### インフラ・CI/CD
- Vercel + GitHub Actions による自動デプロイパイプライン構築
- 本番/ステージング環境の分離
- エラー監視（Sentry等）の導入

### 機能開発ロードマップ管理
- Phase 2以降の機能（シフト管理・給与連携・AI離職予測）の技術選定
- API設計（派遣会社の既存システムとの連携仕様）
- パフォーマンス計測・改善

## 稼働時の成果物例
- 本番用セキュリティ設計書
- データベース設計書（ER図）
- CI/CDパイプライン設定ファイル
- 技術選定記録（ADR: Architecture Decision Records）
